   ```php
           ['id' => 'order_merge', 'name' => '合併訂單', 'description' => '多筆訂單合併出貨', 'category' => 'orders'],
           ['id' => 'batch_operations', 'name' => '批次操作', 'description' => '批量商品上架和管理', 'category' => 'products'],

           ['id' => 'custom_fields', 'name' => '自定義欄位', 'description' => '商品自定義欄位擴充', 'category' => 'products'],
           ['id' => 'multi_images', 'name' => '多圖輪播', 'description' => '商品多圖片上傳和輪播', 'category' => 'products'],
           ['id' => 'export', 'name' => '資料匯出', 'description' => '訂單和出貨資料匯出', 'category' => 'data'],
       ],
   ];
   ```
   For pro features, merge in the current toggle state from wp_options (add 'enabled' key based on toggle state). If Pro is not active, all pro features should have `'enabled' => false`.

2. `get_feature_toggles(): array` -- Reads `buygo_feature_toggles` from wp_options (default: all pro features enabled). Returns associative array `['helper_system' => true, 'order_merge' => true, ...]`.

3. `save_feature_toggles(array $toggles): bool` -- Validates keys against known pro feature IDs, sanitizes to boolean values, saves to `buygo_feature_toggles` wp_option. Returns true on success.

4. `get_license(): array` -- Returns `['key' => string, 'status' => string, 'expires' => string]` by reading three wp_options: `buygo_license_key`, `buygo_license_status` (values: 'inactive'|'active'|'expired'), `buygo_license_expires` (date string or empty). Default: `['key' => '', 'status' => 'inactive', 'expires' => '']`.

5. `save_license(string $key): array` -- Saves license key to `buygo_license_key`. For now (no license server), if key is non-empty, set status to 'active' and expires to empty string. If key is empty, set status to 'inactive'. Saves all three wp_options. Returns the updated license array.

6. `deactivate_license(): array` -- Clears all three wp_options (key='', status='inactive', expires=''). Returns the cleared license array.

7. `is_pro(): bool` -- Currently always returns true (per FEAT-04 decision). Add a comment: "// TODO: 未來串接授權伺服器後改為檢查 license status 和到期日". The method should also check `buygo_license_status` option -- but since we always return true for now, just return true with the TODO comment.

All methods are static (consistent with SettingsService pattern). Use `get_option()` and `update_option()` directly (no encryption needed for these values).
