    <div class="bgo-fm-feature-card">

    feature-management.js（互動邏輯）：
    1. 頁面載入時 → fetch GET /features 取得完整列表
    2. 渲染 Free 功能卡片（toggle 永遠 enabled，永遠 checked）
    3. 渲染 Pro 功能卡片：
       - Pro 未啟用：toggle disabled + 顯示鎖頭圖示
       - Pro 已啟用：toggle enabled，checked 依 API 回傳
    4. 頁面載入時 → fetch GET /features/license 取得授權狀態
    5. 更新授權卡片（Free/Pro badge、到期日、按鈕顯示）
    6. 授權碼驗證 → fetch POST /features/license {key: '...'}
    7. 停用授權 → fetch DELETE /features/license
    8. Toggle 切換 → 收集所有 Pro toggles → fetch POST /features/toggles {toggles: {...}}
    9. 操作中顯示 loading 狀態

    feature-management.css（樣式）：
    - .bgo-fm-license-card：授權卡片（特殊背景色）
    - .bgo-fm-license-status：狀態顯示區
    - .bgo-fm-license-input：輸入區
    - .bgo-fm-feature-list：功能列表容器
    - .bgo-fm-feature-card：單個功能卡片
    - .bgo-fm-toggle：CSS-only toggle 開關
    - .bgo-fm-toggle-slider：toggle 滑桿

    REST API 端點（已完成，只需呼叫）：
    - GET    /wp-json/buygo-plus-one/v1/features
    - GET    /wp-json/buygo-plus-one/v1/features/toggles
    - POST   /wp-json/buygo-plus-one/v1/features/toggles  {toggles: {helper_system: true, ...}}
    - GET    /wp-json/buygo-plus-one/v1/features/license
    - POST   /wp-json/buygo-plus-one/v1/features/license   {key: '...'}
    - DELETE /wp-json/buygo-plus-one/v1/features/license

    注意：
    - Free 功能 toggle 只是視覺展示（永遠 on），不需呼叫 API
    - Pro toggle 切換後立即呼叫 API 儲存（不用「儲存」按鈕）
    - 授權碼輸入後驗證成功 → 自動刷新功能列表（Pro 功能解鎖）
    - 停用授權 → 所有 Pro toggle 變 disabled
    - CSS class 前綴 .bgo-fm-（Feature Management）
