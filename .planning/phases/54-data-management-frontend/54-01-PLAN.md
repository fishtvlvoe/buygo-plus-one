    data-management.js（互動邏輯）：
    1. 子 Tab 切換 → 清空表格 + 自動查詢
    2. 查詢表單提交 → fetch REST API /data-management/query?type=xxx
    3. 渲染結果表格（訂單/商品/客戶各自不同欄位）
    4. 全選/單選 checkbox 控制
    5. 刪除流程：勾選 → 點刪除 → Modal 顯示 → 輸入 DELETE → 確認 → fetch REST API
    6. 客戶編輯：點編輯按鈕 → Modal 顯示 + 填入現有資料 → 修改 → 儲存 → fetch REST API
    7. 分頁：上一頁/下一頁按鈕

    data-management.css（樣式）：
    - .bgo-dm-sub-tabs：子 Tab 樣式（pill 按鈕風格）
    - .bgo-dm-filters：篩選區佈局
    - .bgo-dm-toolbar：工具列（全選 + 刪除按鈕）
    - .bgo-dm-pagination：分頁按鈕
    - .bgo-modal：Modal 遮罩和內容（複用 admin-settings.css 的 modal 樣式）

    REST API 端點（已完成，只需呼叫）：
    - GET  /wp-json/buygo-plus-one/v1/data-management/query?type=orders&date_from=&date_to=&keyword=&page=1&per_page=20
    - POST /wp-json/buygo-plus-one/v1/data-management/delete-orders  {ids:[], confirmation_token:'DELETE'}
    - POST /wp-json/buygo-plus-one/v1/data-management/delete-products
    - POST /wp-json/buygo-plus-one/v1/data-management/delete-customers
    - PUT  /wp-json/buygo-plus-one/v1/data-management/customers/{id}

    表格欄位定義：
    - 訂單：ID, 訂單編號, 客戶, 金額, 狀態, 建立日期
    - 商品：ID, 商品名稱, 價格, 庫存, 狀態, 建立日期
    - 客戶：ID, 姓名, Email, 電話, 訂單數, 建立日期, [編輯]

    注意：
    - 使用 wp.apiFetch 或原生 fetch + wpApiSettings.root + wpApiSettings.nonce
    - Modal 樣式對齊現有的角色權限 Modal（roles-tab.php 中的 .buygo-modal）
    - 刪除按鈕紅色（#d63638），與 developer-tab.php 一致
    - CSS class 前綴 .bgo-dm-（Data Management）
