Method `register_routes()` registers these routes:

1. **POST /products/batch-create** (API-01)
   - permission_callback: `[API::class, 'check_permission']`
   - callback: returns `new \WP_REST_Response(['success' => false, 'message' => 'Batch create is not yet implemented. This endpoint is reserved for future Pro features.', 'code' => 'not_implemented'], 501)`

2. **POST /products/(?P<id>\d+)/images** (API-02)
   - permission_callback: `[API::class, 'check_permission']`
   - callback: returns 501 with message 'Image upload is not yet implemented. This endpoint is reserved for future Pro features.'

3. **GET /products/(?P<id>\d+)/images** (API-02)
   - permission_callback: `[API::class, 'check_permission']`
   - callback: returns 501 with message 'Image listing is not yet implemented. This endpoint is reserved for future Pro features.'

4. **GET /products/(?P<id>\d+)/custom-fields** (API-03)
   - permission_callback: `[API::class, 'check_permission']`
   - callback: returns 501 with message 'Custom fields reading is not yet implemented. This endpoint is reserved for future Pro features.'

5. **PUT /products/(?P<id>\d+)/custom-fields** (API-03)
   - permission_callback: `[API::class, 'check_permission']`
   - callback: returns 501 with message 'Custom fields updating is not yet implemented. This endpoint is reserved for future Pro features.'

Implementation approach -- use a single private method to generate 501 responses:

```php
private function not_implemented(string $feature): \WP_REST_Response
{
    return new \WP_REST_Response([
        'success' => false,
        'message' => "{$feature} is not yet implemented. This endpoint is reserved for future Pro features.",
        'code'    => 'not_implemented',
    ], 501);
}
```

Then each callback method is minimal:
```php
public function batch_create($request)
{
    return $this->not_implemented('Batch create');
}
```

For routes with `(?P<id>\d+)`, add the `id` arg with validate_callback checking is_numeric.

Class doc comment should explain these are placeholder endpoints for Pro features.
