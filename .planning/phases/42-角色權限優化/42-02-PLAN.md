# Plan 42-02: 搜尋 UX 改進

**Phase:** 42 - 角色權限優化
**Goal:** 搜尋框 focus 即顯示前 20 筆用戶，輸入後即時篩選

---

## Context

目前搜尋需要輸入 ≥ 2 字元才觸發 API，空搜尋回傳空陣列。SC #2 要求點擊搜尋框即顯示。

## Tasks

### Task 1: 修改 search_users() API

**File:** `includes/api/class-settings-api.php:361-428`

空 query 時回傳前 20 筆用戶（而非空陣列）：

```php
if (empty($query)) {
    // 空搜尋：回傳前 20 筆用戶
    $args = [
        'orderby' => 'display_name',
        'order' => 'ASC',
        'number' => 20,
    ];
} else {
    $query = substr($query, 0, 50);
    $args = [
        'search' => '*' . $query . '*',
        'search_columns' => ['user_login', 'user_nicename', 'user_email', 'display_name'],
        'number' => 20,
    ];
}
```

### Task 2: 修改前端搜尋行為

**File:** `admin/js/admin-settings.js:60-137`

修改 `setupSearchField()` 函式：
1. 移除「至少輸入 2 個字元」的限制
2. 新增 `focus` 事件：focus 時自動觸發空搜尋
3. 輸入字元後即時篩選（保持 300ms debounce）

```javascript
$input.on('focus', function() {
    if (!$hidden.val()) { // 尚未選取用戶時
        triggerSearch('');
    }
});

$input.on('input', function() {
    const query = $(this).val().trim();
    clearTimeout(timeout);
    timeout = setTimeout(function() {
        triggerSearch(query);
    }, 300);
});
```

### Task 3: 更新提示文字

**File:** `includes/admin/tabs/roles-tab.php:279`

將「至少輸入 2 個字元開始搜尋」改為「點擊搜尋框選擇用戶」
