# Plan 42-03: 新增流程簡化

**Phase:** 42 - 角色權限優化
**Goal:** 「新增賣家」一鍵操作 — 搜尋選擇用戶後直接賦予 buygo_admin 角色

---

## Context

目前「新增角色」Modal 有 3 個欄位（搜尋用戶、選擇角色、歸屬賣家）。SC #3 要求簡化：
- 按鈕改名為「新增賣家」
- 移除角色 dropdown
- 移除歸屬賣家搜尋
- 選擇用戶後直接賦予 buygo_admin

小幫手的新增改為從賣家的「綁定關係」欄位操作（已有 add-helper-modal）。

## Tasks

### Task 1: 修改 Modal HTML

**File:** `includes/admin/tabs/roles-tab.php:260-318`

1. 按鈕文字「新增角色」→「新增賣家」
2. 移除角色 dropdown (`<tr>` 包含 `#add-role-type`)
3. 移除歸屬賣家搜尋 (`<tr id="add-role-seller-row">`)
4. Modal 標題「新增角色」→「新增賣家」

### Task 2: 修改 JS 邏輯

**File:** `admin/js/admin-settings.js:143-245`

1. 移除 `$('#add-role-type').on('change', ...)` 監聽
2. `$('#confirm-add-role')` click handler：
   - 移除 role 和 sellerId 變數
   - payload 固定為 `{ user_id: parseInt(userId), role: 'buygo_admin' }`
   - 移除 sellerId 驗證

### Task 3: 搜尋框整合 42-02 的 focus 行為

新增賣家 Modal 的搜尋框也應用 focus 即顯示的行為（從 42-02 繼承）。

## Verification

- 點擊「新增賣家」→ Modal 只有一個搜尋框
- 搜尋選擇用戶 → 點擊確認 → 用戶獲得 buygo_admin 角色
- 頁面刷新後，新賣家出現在表格中
