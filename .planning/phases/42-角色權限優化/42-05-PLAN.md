# Plan 42-05: API 和 Portal 層權限檢查

**Phase:** 42 - 角色權限優化
**Goal:** API 和 Portal 導航套用 buygo_helper_can() 檢查

---

## Context

42-04 建立了 `SettingsService::helper_can()` 函式。本 Plan 將此函式套用到 Portal 前端導航和 REST API 端點。

## Tasks

### Task 1: Portal 導航權限檢查

**File:** `includes/views/template.php`

在導航選單項目中加入權限判斷：
```php
$current_user_id = get_current_user_id();
$nav_items = [
    'products'  => ['label' => '商品管理', 'permission' => 'products'],
    'orders'    => ['label' => '訂單管理', 'permission' => 'orders'],
    'shipments' => ['label' => '出貨管理', 'permission' => 'shipments'],
    'customers' => ['label' => '客戶管理', 'permission' => 'customers'],
    'settings'  => ['label' => '設定',     'permission' => 'settings'],
];

foreach ($nav_items as $key => $item) {
    if (!SettingsService::helper_can($current_user_id, $item['permission'])) {
        continue; // 跳過無權限的頁面
    }
    // 渲染導航項目
}
```

### Task 2: REST API 權限檢查

**File:** 各 API class 的 permission_callback

為各 API 端點的 `permission_callback` 加入 helper_can 檢查：
- Products API → `helper_can('products')`
- Orders API → `helper_can('orders')`
- Shipment API → `helper_can('shipments')`
- Customers API → `helper_can('customers')`
- Settings API → `helper_can('settings')`（僅 admin 相關端點）

**注意：** 不影響 `manage_options` 的端點（WP Admin 的端點）。

### Task 3: no-access 頁面處理

**File:** `includes/views/no-access.php`（已存在）

當小幫手存取無權限頁面時：
- Portal 路由檢查 → 顯示 no-access.php 的友善提示
- API 回傳 403 + 明確的錯誤訊息

### Task 4: 傳遞權限到前端

**File:** `includes/views/template.php`

在 `wp_localize_script` 中加入當前用戶的權限陣列：
```php
'userPermissions' => [
    'products'  => SettingsService::helper_can($current_user_id, 'products'),
    'orders'    => SettingsService::helper_can($current_user_id, 'orders'),
    'shipments' => SettingsService::helper_can($current_user_id, 'shipments'),
    'customers' => SettingsService::helper_can($current_user_id, 'customers'),
    'settings'  => SettingsService::helper_can($current_user_id, 'settings'),
]
```

前端 Vue RouterMixin.js 可根據此資料隱藏導航項目。

## Verification

- 小幫手關閉「客戶資料」權限後：
  - Portal 導航不顯示「客戶管理」
  - 直接存取客戶管理 URL 顯示 no-access 提示
  - API 呼叫客戶相關端點回傳 403
- 賣家不受影響（所有頁面正常顯示）
