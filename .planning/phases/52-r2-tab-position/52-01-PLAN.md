修改 `register_buygo_tab()` 方法（第 52-55 行），從簡單的陣列附加改為位置插入。

現有程式碼：
```php
public function register_buygo_tab( array $tabs ): array {
    $tabs['r2'] = __( 'R2 圖床', 'bgo-r2' );
    return $tabs;
}
```

改為：
```php
public function register_buygo_tab( array $tabs ): array {
    // 插入到 roles 之後（Tab 順序：角色權限 → R2 圖床 → LINE 模板 → ...）
    $position = array_search( 'roles', array_keys( $tabs ), true );
    if ( $position !== false ) {
        $before = array_slice( $tabs, 0, $position + 1, true );
        $after  = array_slice( $tabs, $position + 1, null, true );
        $tabs   = $before + [ 'r2' => __( 'R2 圖床', 'bgo-r2' ) ] + $after;
    } else {
        // Fallback：如果找不到 roles，附加到最前面
        $tabs = [ 'r2' => __( 'R2 圖床', 'bgo-r2' ) ] + $tabs;
    }
    return $tabs;
}
```

注意：
- 使用 `array_search` + `array_slice` 做位置插入，保留原始 PHP 關聯陣列的鍵值
- `$position + 1` 確保插入在 roles 之後
- Fallback 邏輯確保即使 BuyGo 的 Tab 結構改變，R2 Tab 仍會顯示
- 不改動方法簽名或類別結構，最小改動原則
