  <behavior>
    BatchCreateService::batchCreate(array $items, int $user_id): array

    輸入：
    - $items: 商品資料陣列，每個元素含 title(必填)、price(必填)、quantity(選填,預設0)、description(選填)、currency(選填,預設TWD)
    - $user_id: 賣家或小幫手的 WordPress User ID

    輸出：
    - ['success' => bool, 'total' => int, 'created' => int, 'failed' => int, 'results' => [...]]
    - results 陣列中每筆含 ['index' => int, 'success' => bool, 'product_id' => int|null, 'error' => string|null]

    Cases:
    1. 空陣列 → ['success' => false, 'error' => '商品資料不可為空']
    2. 超過單次上限(50筆) → ['success' => false, 'error' => '單次最多上架 50 個商品']
    3. 配額不足(需要建 5 筆但只剩 2 個配額) → ['success' => false, 'error' => '商品配額不足...']
    4. 某筆缺少 title → 該筆 results: ['success' => false, 'error' => '商品名稱為必填']
    5. 某筆 price < 0 → 該筆 results: ['success' => false, 'error' => '價格不可為負數']
    6. 某筆 FluentCart 建立失敗 → 該筆 results: ['success' => false, 'error' => '...']，其他已成功的不回滾
    7. 全部成功 → ['success' => true, 'total' => N, 'created' => N, 'failed' => 0]

    配額邏輯：
    - 呼叫 ProductLimitChecker::canAddProduct($user_id) 取得 can_add, current, limit
    - 如果 limit > 0（有限制），檢查 current + count($items) <= limit
    - 如果 limit == 0（無限制），跳過配額檢查
    - 配額在批次開始前一次性檢查（不是逐筆檢查）

    注意：BatchCreateService 需要可注入 FluentCartService 和 ProductLimitChecker（方便測試 mock）
  </behavior>
  <implementation>
    1. RED: 寫 BatchCreateServiceTest（7 個測試方法對應上述 7 Cases）
    2. GREEN: 實作 BatchCreateService
       - constructor 接受 FluentCartService 和 ProductLimitChecker（可選，預設 new）
       - batchCreate() 方法：
         a. 驗證 $items 非空、不超過 50 筆
         b. 呼叫 canAddProduct 檢查配額
         c. 逐筆驗證必填欄位
         d. 逐筆呼叫 FluentCartService::create_product()
         e. 收集每筆結果（成功帶 product_id，失敗帶 error）
         f. 彙整回傳
    3. REFACTOR: 確保程式碼符合熵減原則（< 300 行）
  </implementation>
</feature>
