  <files>buygo-plus-one/includes/services/class-data-management-service.php</files>
  <action>
Create `includes/services/class-data-management-service.php` with namespace `BuyGoPlus\Services`.

The class should have these query methods:

1. **queryOrders(array $params): array** — Query fct_orders by date range and keyword
   - Parameters: `date_from` (string Y-m-d), `date_to` (string Y-m-d), `keyword` (string), `page` (int, default 1), `per_page` (int, default 20)
   - Date filter: `WHERE o.created_at BETWEEN %s AND %s` (append 00:00:00 and 23:59:59)
   - Keyword filter: search in `o.invoice_no` and customer name (JOIN fct_customers for `first_name`, `last_name`, `email`)
   - Exclude child orders: `AND o.parent_id IS NULL`
   - Return: `['data' => [...], 'total' => int, 'page' => int, 'per_page' => int]`
   - Each order row: id, invoice_no, customer_name, customer_email, total_amount, currency, status, payment_status, item_count, created_at
   - Use `$wpdb->prepare()` for ALL queries (no raw string interpolation)

2. **queryProducts(array $params): array** — Query FluentCart products by date range and keyword
   - Use `$wpdb->prefix . 'fct_order_items'` joined with `$wpdb->posts` to get products
   - Actually, query `wp_posts` WHERE post_type = 'fc_product' (FluentCart product post type) joined with `{prefix}fct_product_variations` (pv) for price/status
   - Date filter on `wp_posts.post_date`
   - Keyword filter on `wp_posts.post_title`
   - Filter: only active products (pv.item_status != 'inactive')
   - Return: `['data' => [...], 'total' => int, 'page' => int, 'per_page' => int]`
   - Each product row: id (variation_id), post_id, name, price, status (published/private), item_status, created_at

3. **queryCustomers(array $params): array** — Query fct_customers by date range and keyword
   - Date filter on `c.created_at`
   - Keyword filter on `c.first_name`, `c.last_name`, `c.email`
   - Join fct_customer_addresses (is_primary=1) for phone
   - Return: `['data' => [...], 'total' => int, 'page' => int, 'per_page' => int]`
   - Each customer row: id, user_id, email, full_name, phone, order_count (from COUNT on fct_orders), total_spent, created_at

All query methods must use `global $wpdb` and `$wpdb->prepare()`.

<task type="auto">
  <name>Task 2: Add delete and edit methods to DataManagementService</name>
  <files>buygo-plus-one/includes/services/class-data-management-service.php</files>
  <action>
Add the following methods to the DataManagementService class created in Task 1:

4. **deleteOrders(array $order_ids): array** — Hard delete orders with cascade cleanup
   - Validate all IDs are positive integers
   - For each order_id:
     a. Delete from `buygo_shipment_items` WHERE order_id = %d
     b. Find shipment_ids that have NO remaining items in buygo_shipment_items, delete those from `buygo_shipments`
     c. Delete from `fct_order_items` WHERE order_id = %d
     d. Delete from `fct_order_addresses` WHERE order_id = %d
     e. Delete child orders: find child order IDs from fct_orders WHERE parent_id = %d, then recursively delete their items/addresses
     f. Delete from `fct_orders` WHERE id = %d OR parent_id = %d
   - Wrap entire operation in `$wpdb->query('START TRANSACTION')` / `COMMIT` / `ROLLBACK` on error
   - Return: `['success' => true, 'deleted_count' => int, 'deleted_ids' => array, 'cascade' => ['shipment_items' => int, 'shipments' => int, 'order_items' => int, 'child_orders' => int]]`
   - WHY transaction: Partial delete of orders with orphaned shipment items is worse than no delete

5. **deleteProducts(array $product_ids): array** — Soft delete products (match existing pattern)
   - Use the same pattern as Products_API::batch_delete(): set `FluentCart\App\Models\ProductVariation::item_status = 'inactive'`
   - Do NOT hard delete — this matches the existing project convention
   - Return: `['success' => true, 'deleted_count' => int, 'failed_ids' => array]`

6. **deleteCustomers(array $customer_ids): array** — Hard delete customers
   - For each customer_id:
     a. Delete from `fct_customer_addresses` WHERE customer_id = %d
     b. Delete from `fct_customers` WHERE id = %d
   - Do NOT delete the WordPress user account (wp_users) — only FluentCart customer data
   - Do NOT delete the customer's orders (orders remain as orphaned records for accounting)
   - Use transaction
   - Return: `['success' => true, 'deleted_count' => int, 'deleted_ids' => array]`

7. **updateCustomer(int $customer_id, array $data): array** — Edit customer record
   - Accepted fields in $data: `first_name`, `last_name`, `phone`, `address_1`, `address_2`, `city`, `state`, `postcode`, `country`, `taiwan_id_number`
   - Update fct_customers: first_name, last_name, updated_at
   - Update fct_customer_addresses (is_primary=1): phone, address_1, address_2, city, state, postcode, country
   - If taiwan_id_number is provided AND the customer has a user_id, update wp_usermeta key `buygo_taiwan_id_number`
   - Sanitize all text inputs with sanitize_text_field()
   - Return: `['success' => true, 'customer_id' => int]` or `['success' => false, 'message' => string]`

IMPORTANT: For deleteOrders, the cascade order matters. Delete shipment_items BEFORE shipments, delete order_items BEFORE orders. Foreign key integrity requires this specific order.
