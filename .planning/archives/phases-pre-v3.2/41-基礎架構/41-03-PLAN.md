# Plan 41-03: Tab 框架整合

**Phase:** 41 - 基礎架構
**Goal:** 將現有 Tab 內容正確掛載到新 6-Tab 框架，確保功能完整不中斷

---

## Context

41-01 建立了 `render_page()` 統一渲染函式和 6-Tab switch 邏輯。
41-02 建立了 `.bgo-tabs` CSS。

本 Plan 需要：
1. 確保 3 個已有 Tab（角色權限、LINE 模板、結帳設定）在新框架中正常渲染
2. 處理 LINE 模板的表單提交邏輯（原本在獨立頁面，現在在 Tab 中）
3. 開發者 Tab 暫時掛載 workflow-tab.php（Phase 45 才合併三工具）
4. 清理不再使用的舊 render 方法引用

## Tasks

### Task 1: 驗證 templates-tab.php 在新框架中的行為

**File:** `includes/admin/tabs/templates-tab.php`

模板 Tab 原本在獨立頁面渲染（`render_templates_page()`）。需確認：

1. 表單 action 是否依賴 `$_GET['page']`：
   - 模板表單使用 `$_POST['submit_templates']`，由 `render_page()` 的表單處理邏輯接住
   - `render_page()` 已包含 `handle_templates_submit()` 處理

2. 模板 Tab 的 3 個子 Tab（買家/賣家/系統）使用 jQuery JS 切換，不依賴 URL 參數，不受影響

3. CSS：模板 Tab 的樣式寫在 `templates-tab.php` 內部的 `<style>` 標籤中（inline），不依賴外部 CSS

**動作：** 讀取 `templates-tab.php` 確認無 hardcoded `?page=buygo-settings` 或 `?page=buygo-plus-one` 的 form action。如有，改為 `?page=buygo-plus-one&tab=templates`。

### Task 2: 驗證 checkout-tab.php 的表單提交

**File:** `includes/admin/tabs/checkout-tab.php`

結帳設定使用 `$_POST['submit']` 表單提交。確認：
- form action 指向正確頁面
- nonce 驗證不受 Tab 切換影響
- 提交後回到正確的 Tab（可能需要在 form 中加 hidden field `tab=checkout`）

**動作：** 確認表單 action 和 redirect 邏輯。如果表單 submit 後會跳回預設 Tab，需要在 form 中加入：
```html
<input type="hidden" name="tab" value="checkout">
```
並在 `render_page()` 的表單處理後設定 `$current_tab`：
```php
if (isset($_POST['tab'])) {
    $current_tab = sanitize_text_field($_POST['tab']);
}
```

### Task 3: 處理開發者 Tab 的臨時內容

**File:** `includes/admin/class-settings-page.php`

在 `render_page()` 的 switch case 中：
- `developer` Tab 暫時載入 `render_workflow_tab()`（流程監控）
- 顯示一個提示：「測試工具和除錯中心將在後續版本中整合到此 Tab」

```php
case 'developer':
    echo '<div class="bgo-card" style="background: #fff3cd; border-color: #ffc107;">';
    echo '<p><strong>開發者工具</strong> — 測試工具和除錯中心將在後續版本中整合到此 Tab。</p>';

    $this->render_workflow_tab();
    break;
```
