  <action>
    在 Reserved_API 中替換 batch_create 方法，從 501 骨架改為真實邏輯：

    1. **權限升級**：將 batch-create 路由的 permission_callback 從 `API::check_permission` 改為使用 products scope 檢查：
       ```php
       'permission_callback' => function() {
           return API::check_permission_with_scope('products');
       },
       ```
       這確保小幫手必須有「商品管理」權限才能批量上架。

    2. **替換 batch_create 方法**：
       ```php
       public function batch_create($request) {
           $body = $request->get_json_params();
           $items = $body['items'] ?? $body;  // 支援 {items: [...]} 或直接傳陣列

           // 確保 items 是陣列
           if (!is_array($items)) {
               return new \WP_REST_Response([
                   'success' => false,
                   'message' => '請求格式錯誤，需要 JSON 陣列',
                   'code' => 'invalid_format',
               ], 400);
           }

           $user_id = get_current_user_id();
           $service = new \BuyGoPlus\Services\BatchCreateService();
           $result = $service->batchCreate($items, $user_id);

           // 根據結果決定 HTTP 狀態碼
           if (!$result['success'] && isset($result['error'])) {
               // 整批失敗（配額不足、空陣列等）
               $status = 422;
               if (strpos($result['error'], '配額') !== false) {
                   $status = 403;
               }
               return new \WP_REST_Response($result, $status);
           }

           // 部分成功或全部成功
           return new \WP_REST_Response($result, 200);
       }
       ```

    3. **更新類別頂部註解**：將 batch-create 從「預留端點」列表移到「已實作端點」列表。

    4. **在 register_routes() 中更新路由註冊**：
       - 新增 args 驗證：
         ```php
         'args' => [
             'items' => [
                 'required' => false,  // 支援直接傳陣列或 {items: [...]}
                 'type' => 'array',
             ],
         ],
         ```

    注意：不要動其他端點（images 仍為 501），只改 batch-create 相關部分。
