# Plan 42-01: 表格過濾邏輯

**Phase:** 42 - 角色權限優化
**Goal:** 角色權限表格只顯示有 BGO 角色的用戶

---

## Context

目前 roles-tab.php 查詢 `administrator` + `buygo_admin` + `buygo_helper`，導致純 WP Admin（無 BGO 角色）也出現在表格中。

## Tasks

### Task 1: 修改 roles-tab.php 的查詢邏輯

**File:** `includes/admin/tabs/roles-tab.php:9-12`

移除 `get_users(['role' => 'administrator'])`，改為：

```php
// 只查詢有 BGO 角色的使用者
$buygo_admins = get_users(['role' => 'buygo_admin']);
$buygo_helpers = get_users(['role' => 'buygo_helper']);
$all_related_users = array_merge($buygo_admins, $buygo_helpers);
```

### Task 2: 移除 $wp_admin_ids 的判斷

原本用 `$wp_admin_ids` 來標記「WordPress 管理員無法移除」。新邏輯：
- 如果用戶同時是 WP Admin + BGO Admin，可以移除 BGO Admin 角色（但保留 WP Admin）
- 移除「WordPress 管理員無法移除」的限制，改為所有人都可以移除 BGO 角色

### Task 3: 調整「角色」欄位顯示

- `buygo_admin` → 顯示「賣家」（不再顯示 BuyGo 管理員）
- `buygo_helper` → 顯示「小幫手」
- WP Admin 標記用小 badge 表示

### Task 4: 操作欄位調整

所有 BGO 角色的用戶都顯示「移除角色」按鈕，不論是否為 WP Admin。
移除角色只移除 BGO 角色，不影響 WP Admin 角色。

## Verification

- 純 WP Admin（無 BGO 角色）不出現在表格中
- WP Admin + BGO Admin 的用戶出現，且可以移除 BGO Admin 角色
- 移除 BGO 角色後，該用戶從表格消失（頁面刷新後）
