  <action>
在 useBatchCreate.js 的 CSV 匯入區塊之後、導航區塊之前，新增「提交與結果」區塊。具體實作：

1. **新增狀態 ref:**
   - `submitting = ref(false)` — 提交中狀態（控制按鈕 disabled + spinner）
   - `submitError = ref('')` — 全局提交錯誤訊息（網路錯誤等）

2. **新增 computed:**
   - `validItems` — 篩選 items 中 name.trim() 非空且 price.trim() 非空且 Number(price) > 0 的商品
   - `validItemCount` — validItems.value.length

3. **新增 submitBatch 方法（async）:**
   - 防重複：if (submitting.value) return
   - 前置驗證：if (validItemCount.value === 0) return
   - 設定 submitting.value = true，清空 submitError
   - 構建 payload：`{ items: validItems.value.map(item => ({ name: item.name.trim(), price: item.price.trim(), quantity: item.quantity || '0', description: item.description.trim() })) }`
   - 呼叫 `const { post } = useApi()`（在 composable 頂部已有 `const { get } = useApi()`，改為 `const { get, post } = useApi()`）
   - try/catch 呼叫 `post('/wp-json/buygo-plus-one/v1/products/batch-create', payload, { showError: false, showSuccess: false })`
   - **全部成功**（res.data.failed === 0）：
     - 用 window.showToast 顯示成功 toast（`'成功上架 ' + res.data.created + ' 個商品'`, 'success'）
     - 用 setTimeout 延遲 800ms 後 goBack()（跳回商品列表頁）
   - **部分失敗**（res.data.created > 0 && res.data.failed > 0）：
     - window.showToast 顯示摘要 toast（`'成功 ' + res.data.created + ' 個，失敗 ' + res.data.failed + ' 個'`, 'error'）
     - 從 res.data.results 中找出失敗的 index（result.success === false）
     - 把成功的 items 從 items.value 中移除（只保留失敗的）
     - 在保留的 items 上標記錯誤：item._error = result.error（新增 _error 屬性）
   - **全部失敗**（res.data.created === 0 或 catch 區塊）：
     - window.showToast 顯示錯誤 toast（`'上架失敗：' + (res.data.results?.[0]?.error || '請檢查商品資料')`, 'error'）
     - 標記所有 items 的 _error
   - catch 區塊（網路錯誤 / API 完全無回應）：
     - submitError.value = err.message || '網路錯誤，請稍後重試'
     - window.showToast 顯示錯誤 toast
   - finally：submitting.value = false

4. **新增 clearItemErrors 方法:**
   - 遍歷 items.value，刪除每個 item 的 _error 屬性
   - 在 submitBatch 開頭呼叫（重試前清除舊錯誤）

5. **擴充 return 物件:**
   在 return 中新增一個「提交（Phase 59）」註解區塊：
   ```
   // 提交（Phase 59）
   submitting, submitError, validItems, validItemCount,
   submitBatch, clearItemErrors,
   ```

**注意事項：**
- useApi() 呼叫改為解構 get 和 post：`const { get, post } = useApi();`
- items 存字串型態（延續慣例），提交時 price 直接傳字串讓後端轉型
- _error 是臨時屬性，不影響表單提交 payload（只在 validItems 的 map 中排除）
- 部分失敗時的 index 對應：API 回傳的 results[i].index 對應 payload.items 的 index，需對應回 validItems 再對應回 items
