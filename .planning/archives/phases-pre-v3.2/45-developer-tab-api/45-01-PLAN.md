**Section 3: SQL Query Console (DEV-03)**
Wrap in `.bgo-card`:
- Card title: `<h3>SQL Console (SELECT Only)</h3>`
- Textarea for SQL input (6 rows, full width, monospace font)
- "Execute" button
- Results area (initially hidden)
- Uses WordPress AJAX (wp_ajax action `buygo_dev_sql_query`)
- **Security**: Server-side validation that query starts with SELECT (case-insensitive, trimmed). Block INSERT, UPDATE, DELETE, DROP, ALTER, CREATE, TRUNCATE, REPLACE, GRANT, REVOKE.
- The AJAX handler:
  1. Verify nonce
  2. Check manage_options permission
  3. Trim and validate query starts with SELECT
  4. Execute via `$wpdb->get_results($sql, ARRAY_A)`
  5. Return results as JSON (limit 100 rows)
- JavaScript renders results as an HTML table
- Show row count and execution time

**Inline JavaScript** at the bottom of the file handles:
1. Data cleanup AJAX call with confirmation
2. SQL query AJAX call with result rendering

**AJAX handlers** should be registered in the same file using `add_action('wp_ajax_...')` at the top of the file (after ABSPATH guard), following WordPress AJAX conventions. Each handler:
- Checks nonce via `check_ajax_referer('buygo_dev_nonce', 'nonce')`
- Checks `current_user_can('manage_options')`
- Returns via `wp_send_json_success()` or `wp_send_json_error()`

**Note on localize_script**: Since admin-settings.js is already enqueued with buygoSettings.ajaxUrl and buygoSettings.nonce, the developer tab JavaScript can reference `buygoSettings.ajaxUrl`. However, for the developer-specific nonce, output it inline via PHP:

```php
</script>
```
