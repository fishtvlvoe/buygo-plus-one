  <name>Task 1: Create FeatureManagement_API REST endpoints</name>
  <files>includes/api/class-feature-management-api.php</files>
  <action>
Create `includes/api/class-feature-management-api.php` following the exact same pattern as `class-data-management-api.php`.

Namespace: `BuyGoPlus\Api`
Use statement: `use BuyGoPlus\Services\FeatureManagementService;`

Class: `FeatureManagement_API` with `private $namespace = 'buygo-plus-one/v1';`

Method `register_routes()` registers these routes:

1. **GET /features** -- Get feature list
   - permission_callback: `[$this, 'check_permission_for_admin']`
   - callback: `get_features` -- calls `FeatureManagementService::get_features()`, returns `['success' => true, 'data' => $features]` with HTTP 200
   - No args needed

2. **GET /features/toggles** -- Get current toggle states
   - permission_callback: `[$this, 'check_permission_for_admin']`
   - callback: `get_toggles` -- calls `FeatureManagementService::get_feature_toggles()`, returns `['success' => true, 'data' => $toggles]`

3. **POST /features/toggles** -- Save toggle states
   - permission_callback: `[$this, 'check_permission_for_admin']`
   - callback: `save_toggles` -- reads JSON body, extracts `toggles` array, calls `FeatureManagementService::save_feature_toggles($toggles)`. Returns `['success' => true, 'data' => FeatureManagementService::get_feature_toggles()]` on success, or `['success' => false, 'message' => ...]` with HTTP 400 on invalid input.
   - args: `toggles` required, validate_callback checks it's an array

4. **GET /features/license** -- Get license status
   - permission_callback: `[$this, 'check_permission_for_admin']`
   - callback: `get_license` -- calls `FeatureManagementService::get_license()`, returns `['success' => true, 'data' => $license]`

5. **POST /features/license** -- Save/activate license
   - permission_callback: `[$this, 'check_permission_for_admin']`
   - callback: `save_license` -- reads JSON body, extracts `key` string, calls `FeatureManagementService::save_license($key)`. Returns `['success' => true, 'data' => $result]`.
   - args: `key` required, sanitize_callback: sanitize_text_field

6. **DELETE /features/license** -- Deactivate/clear license
   - permission_callback: `[$this, 'check_permission_for_admin']`
   - callback: `deactivate_license` -- calls `FeatureManagementService::deactivate_license()`. Returns `['success' => true, 'data' => $result]`.

Method `check_permission_for_admin(): bool` -- exact same implementation as DataManagement_API:
```php
public function check_permission_for_admin()
{
    if (!is_user_logged_in()) {
        return false;
    }
    return current_user_can('buygo_admin') || current_user_can('manage_options');
}
```

All callback methods should wrap logic in try/catch, returning HTTP 500 with error message on exception (same pattern as DataManagement_API).

For POST endpoints that read JSON body, use: `$body = json_decode($request->get_body(), true);` then extract fields -- same pattern as DataManagement_API delete methods.
