# AI Writer - 自動化寫作助手

## 何時使用此 Skill

當使用者需要：
- 撰寫行銷文案（社群貼文、部落格、Email）
- 產出多種平台的內容
- 需要結構化的寫作流程
- 想要逐步審核文章品質

**觸發詞**：寫文章、寫貼文、寫文案、寫 Email、寫部落格、ai-writer

## 支援的平台

| 平台 | 建議字數 | 特色 |
|-----|---------|-----|
| 部落格 | 1000-2000 | SEO 優化、H2/H3 分段 |
| Facebook | 100-300 | 情緒化開頭、互動問句 |
| Instagram | 50-150 | 短句、hashtag |
| LinkedIn | 200-500 | 專業語氣、數據佐證 |
| Email | 300-600 | 主旨行、明確 CTA |

## 快速開始

```
/ai-writer [主題] [平台]

範例：
/ai-writer 咖啡的好處 FB
/ai-writer 新品上市公告 Email
/ai-writer 遠端工作心得 部落格
```

## 輸入格式

Skill 會自動判斷輸入類型：

| 輸入長度 | 判斷為 | 處理 |
|---------|-------|-----|
| 1-5 字 | 關鍵字 | 詢問補充資訊 |
| 6-50 字 | 簡短描述 | 直接進入大綱 |
| 50+ 字 | 完整 Brief | 解析後進入大綱 |

## 工作流程（5 階段）

```
輸入解析 → 大綱生成 → 草稿撰寫 → 定稿潤飾 → 儲存輸出
           [審核]      [審核]      [審核]     [詢問]
```

每個審核點都會暫停，等待使用者確認或修改。

### 審核選項

**大綱審核：**
- ✅ 確認，開始寫作
- ✏️ 修改某個段落
- 🔄 重新生成

**草稿審核：**
- ✅ 滿意，進入定稿
- ✏️ 修改特定段落
- 🎨 調整語氣風格
- 📏 調整長度

**定稿審核：**
- ✅ 完成
- ✏️ 微調
- 💾 儲存到檔案

## 輸出格式

所有文章以 **Markdown** 格式輸出，可直接複製使用。

儲存時檔名格式：`YYYY-MM-DD-[主題]-[平台].md`

## 執行指令

當此 Skill 被觸發時，請遵循以下步驟：

### 步驟 1：解析輸入

1. 判斷輸入長度和類型
2. 如果資訊不足，使用 AskUserQuestion 詢問：
   - 目標平台
   - 目標讀者
   - 文章目的（導購/品牌/教育）

### 步驟 2：生成大綱

1. 參考 TEMPLATES.md 選擇對應平台模板
2. 生成結構化大綱，包含：
   - 開頭 Hook
   - 主體段落（2-4 個重點）
   - 結尾 CTA
3. 顯示大綱，等待使用者確認

### 步驟 3：撰寫草稿

1. 參考 STYLES.md 套用適當語氣
2. 根據大綱撰寫完整文章
3. 顯示草稿和字數統計
4. 等待使用者審核

### 步驟 4：潤飾定稿

1. 根據使用者意見修改
2. 檢查錯別字、標點
3. 優化開頭吸引力
4. 強化 CTA
5. 顯示定稿

### 步驟 5：儲存輸出

1. 詢問是否儲存到檔案
2. 如果是，詢問儲存位置（預設 `./articles/`）
3. 儲存為 Markdown 檔案

**重要**：每個審核點都必須暫停，等待使用者明確確認後才能繼續。
