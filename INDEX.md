## 🔄 開發工作流程

### 每日流程

```
1. 檢查系統狀態（30 分鐘）
   ├── 開啟正式環境，確認運行正常
   └── 記錄發現的問題

2. 回報問題給 Claude（10 分鐘）
   ├── 使用標準格式描述問題
   └── 或使用 /debug-buygo 快速診斷

3. Claude 修復 + 你做其他事（1-2 小時）
   ├── Claude 分析和修復
   └── 你可以處理其他工作

4. 測試修復結果（30 分鐘）
   ├── 按照 Claude 給的測試清單
   └── 回報結果（OK 或有問題）

5. 確認完成或繼續修復
   └── 進入下一個任務
```

### 問題回報格式

```
【問題】簡短描述
【頁面】在哪個頁面發生
【步驟】如何重現
【預期】應該發生什麼
【實際】實際發生什麼
```

### 確認完成格式

```
【測試項目】測試了什麼
【結果】✅ OK 或 ❌ 有問題
【備註】其他說明
```

## 🛡️ 防止 Bug 復發的機制

### 1. 修改前檢查（CLAUDE.md）

Claude 每次修改代碼前，必須確認：
- [ ] 不會破壞 BUGFIX-CHECKLIST.md 中的已修復項目
- [ ] 遵守 CSS/JS 命名規範
- [ ] wpNonce 和 API 權限正確

### 2. 修改後驗證

每次修改後，必須測試：
- [ ] 頁面正常載入
- [ ] API 無 401/500 錯誤
- [ ] 搜尋和分頁正常
- [ ] 核心流程正常

### 3. Debug Skill（/debug-buygo）

遇到問題時，使用 `/debug-buygo` 快速診斷，包含：
- 常見問題的診斷步驟
- 修復方案和關鍵代碼
- 驗證清單
