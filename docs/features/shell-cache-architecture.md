# 殼快取架構計畫 — 買家與賣家分離載入

## 核心概念

把「殼」（固定的 HTML/CSS/JS 框架）跟「資料」（需要登入才能查的內容）分開。
殼可以被 Cloudflare 快取，資料透過 API 載入。

```
現在的載入流程（每次都要等 PHP）：

  瀏覽器 → Cloudflare → InstaWP PHP（2.5 秒）
                         ├─ 組裝 HTML 殼
                         ├─ 查資料庫
                         └─ 回傳完整頁面

改成（殼快取 + API 資料分離）：

  瀏覽器 → Cloudflare（殼已快取，0.1 秒回傳）
           ├─ 立刻顯示殼（側邊欄 + 骨架屏）
           └─ JS 打 API 拿資料（0.5 秒）
              └─ 資料填入殼，畫面完成
```

## 賣家點「會員中心」的流程

```
賣家在 /buygo-portal/ 點側邊欄底部的「會員中心」
  │
  ▼
跳到 /my-account/（FluentCart 會員中心）
  │
  ├─ 看到跟買家一樣的會員中心
  │  （購買歷史、訂單進度、個人資料）
  │
  └─ 但賣家可以用瀏覽器「上一頁」回到 /buygo-portal/
```

## 要做的事（按優先順序）

### Phase 1：login_redirect 分流（最簡單，效果最大）

```php
add_filter('login_redirect', function($redirect_to, $requested, $user) {
    if (is_wp_error($user)) return $redirect_to;

    if (array_intersect(['administrator', 'buygo_admin', 'buygo_helper'], $user->roles)) {
        return home_url('/buygo-portal/dashboard/');
    }

    return home_url('/my-account/');
}, 10, 3);
```

- 賣家登入 → `/buygo-portal/`
- 買家登入 → `/my-account/`
- 不經過 `/buygo-portal/` 再 redirect，省一次跳轉

### Phase 2：殼快取

把賣家後台的 HTML 殼（側邊欄 + 骨架屏 + JS/CSS）做成可以被 Cloudflare 快取的靜態頁面。

做法：
1. PHP 只輸出殼（不查任何資料庫）
2. 登入驗證改到 JS 端做（API 呼叫時帶 cookie）
3. Cloudflare Page Rule：`/buygo-portal/*` → Cache Everything
4. 殼的 HTML 快取後，所有賣家打開都是同一個殼，資料再透過 API 各自載入

風險：
- 殼裡面不能有任何個人化的內容（用戶名、nonce）
- nonce 要改成 API 端點動態取得
- 需要改動 template.php 的架構

### Phase 3：API 預載 + SPA 路由快取

- Service Worker 快取 API 回應
- SPA 頁面切換時，先顯示上次的快取資料，背景更新

## 時間估計

| Phase | 內容 | 工時 | 效果 |
|-------|------|------|------|
| 1 | login_redirect 分流 | 15 分鐘 | 省一次跳轉 |
| 2 | 殼快取 | 3~4 小時 | 0.1 秒看到殼 |
| 3 | API 預載 | 2~3 小時 | SPA 切換更快 |

## 不需要改的部分

- FluentCart 會員中心（不動）
- `fluent_cart_api()` 注入的訂單進度（不動）
- REST API 端點（不動）
- 資料庫結構（不動）
- LINE 通知功能（不動）
